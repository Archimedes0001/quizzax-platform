Phase 1: Project Charter

1.1 Project Title and Description

Project Title: Quizzax Assessment Platform

Brief Description: Quizzax is a secure, web-based assessment platform designed for engineering students. It simplifies the process of taking continuous assessment (CA) quizzes by providing a responsive, interactive interface that offers instant feedback, detailed performance tracking, and a competitive leaderboard system. The platform replaces manual or paper-based testing with a modern, automated solution.

1.2 Project Objectives and Scope

Objectives

  • Automation: To automate the delivery and grading of engineering quizzes.
  • Security: To ensure only authorized students (verified by matric number) can access assessments.
  • Engagement: To increase student engagement through immediate feedback and gamified elements.
  • Analytics: To provide students and instructors with actionable data on academic performance.

Phase 2: System Architecture

1. Architecture Overview

Quizzax employs a 3-Tier Client-Server Architecture to ensure separation of concerns, scalability, and maintainability.

  1. Presentation Tier (Client): Handles UI rendering and user interactions using Vanilla JavaScript.
  2. Application Tier (Server): Processes business logic, authentication, and scoring algorithms (Node.js/Express).
  3. Data Tier (Database): Stores persistent data (users, quizzes, history) in MongoDB Cloud.

2. Technology Stack Selection

Component Technology Justification
Frontend Vanilla JS + HTML5 Lightweight, no build step, high performance.
Backend Node.js & Express Non-blocking I/O for concurrent quiz submissions.
Database MongoDB Schema flexibility for JSON-like quiz documents.
Hosting Render Free tier availability and native Node.js support.

Normalization Explanation

The database design follows 2nd Normal Form (2NF) adapted for NoSQL. We intentionally denormalize the `history` and `questions` arrays by embedding them. This optimizes read performance, allowing the system to fetch a full quiz or student profile in a single database query, crucial for meeting the < 200ms response time requirement.

Phase 3: Implementation Report

Key Algorithms

Fisher-Yates Shuffle

To ensure fairness, we implemented the Fisher-Yates Shuffle algorithm to randomize the question array order for every student.

function shuffle(array) {
  for (let i = array.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [array[i], array[j]] = [array[j], array[i]];
  }
}

Phase 5: Deployment Guide

Render.com Deployment Instructions

  1. Create Account: Sign up at dashboard.render.com.
  2. New Web Service: Connect your GitHub repository `quiz_app`.
  3. Settings:
    • Build Command: npm install
    • Start Command: npm start
  4. Environment Variables:
    • Add MONGO_URI (Value: mongodb+srv://...)
    • Add NODE_ENV (Value: production)
  5. Deploy: Click "Create Web Service".

User Manual: Getting Started

  1. Login: Access URL -> Enter Matric No -> Click "Sign In".
  2. Dashboard: Select "Engineering Math" or "Fluid Mechanics".
  3. Quiz: Answer 50 questions within 15 mins. Use "Flag" to mark for review.
